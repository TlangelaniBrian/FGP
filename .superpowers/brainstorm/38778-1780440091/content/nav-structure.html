How should the app be structured?

You need two main modes: Feasibility analysis (manual entry → score → PDF) and Projects (Soshanguve build tracker). How should navigation divide them?

A

Two top-level sections

Scout (find + analyse land) and Projects (active builds) sit as equal siblings in the sidebar. Scout covers feasibility. Projects covers the Soshanguve tracker and any future build.

Dashboard hub with deep-links

A single Dashboard home page surfaces both live project status (Soshanguve) and recent feasibility analyses side-by-side. Individual sections are reachable from the dashboard only — no persistent sidebar sections.

Projects-first, analysis as a tool inside

Projects are the primary object. Feasibility analysis lives inside a project (or as a pre-project step to evaluate land before committing). The sidebar shows projects; analysis is triggered from within.
